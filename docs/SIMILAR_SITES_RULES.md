# 유사·경쟁 사이트 후보 — 정의 규칙

리포트 **「유사·경쟁 사이트 검색」** 블록에 표시되는 후보는, 외부 검색·크롤 API가 아니라 **`lib/services/ai.ts`의 `findSimilarSites`**가 **Google Gemini**에 보내는 프롬프트 규칙과, 응답을 받은 뒤 적용하는 **코드 후처리**로 결정됩니다.

구현 위치: `findSimilarSites` (동일 파일 상단 `SimilarSite` 타입 참고).

---

## 1. 입력(모델에 넘기는 정보)

후보를 “정의”하기 전에 모델은 아래 세 가지만 받습니다.

| 입력 | 출처 |
|------|------|
| 분석 대상 **URL** | 사용자가 분석에 넣은 페이지 URL |
| **페이지 요약** (`contentSummary`) | 같은 파이프라인의 `analyzeContentInsights`(Gemini) 결과 |
| **주요 타겟층** (`targetAudience`) | 동일 |

즉, **현재 페이지 텍스트 전체가 아니라**, 이미 한 번 요약된 문장과 타겟 설명에 기대어 후보를 제안합니다.

---

## 2. 프롬프트 상의 “유사·경쟁” 정의

모델에게 요청하는 역할은 다음과 같이 고정되어 있습니다.

- **목적·타겟이 비슷한 실제 경쟁사 또는 대체재**를 최대 **3개** 제시할 것.

---

## 3. 필수 조건 (프롬프트)

| 규칙 | 설명 |
|------|------|
| 동일 사이트 제외 | 분석 대상과 **같은 사이트**는 넣지 말 것. **같은 도메인**이거나 **리다이렉트만 다른 경우**도 동일로 간주. |
| URL 형식 | 실제 접속 가능한 **공식 `https` URL**만. 존재하지 않거나 **확실하지 않은 도메인**은 넣지 말 것. |
| 후보 성격 | **잘 알려진 브랜드·서비스** 위주(한국·글로벌 무관). 확신이 없으면 **3개 미만**으로 줄여도 됨. |

---

## 4. 금지 사항 (프롬프트)

| 규칙 | 설명 |
|------|------|
| 허구 금지 | **가상의 회사·URL**을 만들거나, **추측으로 만든 경로**를 넣지 말 것. |

---

## 5. 출력 형식 (프롬프트 + 코드)

### 5.1 모델이 내보내야 하는 JSON

- 최상위 키: `sites` — 배열.
- 각 요소 필드(모두 **문자열**):
  - `url` — `https://...`
  - `name` — 사이트명
  - `matchReason` — **목적·타겟 유사 이유** 한 문장
  - `fameReason` — **규모·인지도** 한 문장
- 항목 개수: **0~3개**.

### 5.2 코드에서 하는 후처리

응답 전체에서 **첫 번째 JSON 객체**를 정규식으로 잘라 파싱합니다.

| 단계 | 동작 |
|------|------|
| 개수 상한 | 배열이 있으면 **`slice(0, 3)`** 으로 최대 3건만 사용. |
| URL 필터 | `url`이 **`http`로 시작**하는 항목만 남김 (`startsWith('http')`). `https`뿐 아니라 `http`도 통과(프롬프트는 https 권장). |
| 실패 시 | 파싱 실패·빈 배열·조건 불충족 시 **`null`** 반환 → 리포트에 `similarSites` 없음. |

프롬프트 문구에는 “목적·타겟 일치도 + 기업 규모/유명도로 **점수화**해 상위 3개”라고 주석에 적혀 있으나, **실제 코드는 별도 점수 계산이 없고**, 모델이 준 순서의 앞 3개를 자릅니다.

---

## 6. 기술적 한계 (문서·사용자 안내용)

| 항목 | 설명 |
|------|------|
| 검색/검증 없음 | **웹 검색·실시간 크롤·링크 유효성 검사**를 하지 않습니다. |
| 지식 시점 | 후보는 **모델의 학습 데이터·추론**에 의존하므로, 최신 서비스·URL이 틀리거나 없을 수 있음. |
| “경쟁”의 의미 | 법적·시장 정의의 경쟁사가 아니라, 프롬프트상 **목적·타겟이 비슷한 대안**에 가깝습니다. |

---

## 7. 호출 조건

`app/api/analyze/route.ts`에서 **`contentSummary`와 `targetAudience`가 둘 다 있을 때만** `findSimilarSites`를 호출합니다. 앞 단계 인사이트가 실패하면 유사 사이트 블록도 나오지 않습니다.

---

## 8. 코드와 문서

규칙이 바뀌면 **프롬프트 문자열**(`findSimilarSites` 내부)과 이 문서를 함께 맞추는 것이 좋습니다. 불일치 시 **코드가 우선**입니다.
